# i18n (Internationalisation)

Part of [[Architecture Overview]].

---

## Supported Locales

| Code | Language |
|---|---|
| `en` | English |
| `ms` | Bahasa Malaysia |

---

## Architecture

**Source:** `lib/i18n/`

| File | Purpose |
|---|---|
| `index.ts` | `getDictionary()` — loads the active locale's dictionary |
| `locale.ts` | Reads/writes locale from cookie |
| `context.tsx` | React context provider for client components |
| `dictionaries/en.ts` | English strings |
| `dictionaries/ms.ts` | Bahasa Malaysia strings |

---

## How It Works

1. Active locale stored in a cookie (set via `lib/actions/locale.ts`).
2. Server Components call `getDictionary()` to get the translation object (`t`).
3. Client Components read locale from `LocaleContext` (provided in `app/layout-wrapper.tsx`).

---

## Locale Toggle

**Component:** `app/components/locale-toggle.tsx`

Appears in the nav header. Clicking it calls the `setLocale` server action and refreshes the page.

---

## Usage Pattern

```ts
// Server Component
const t = await getDictionary();
<h1>{t.dashboard.title}</h1>

// Client Component
const { t } = useLocale();
<button>{t.common.approve}</button>
```
